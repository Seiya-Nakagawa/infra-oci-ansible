# 要件定義書 - infra-oci-ansible

## 1. システムの目的

本プロジェクトは、OCI（Oracle Cloud Infrastructure）上で稼働する個人開発サーバのOSおよびミドルウェア（MW）の構成管理を目的とする。
複数のサービスを効率的に運用し、再現性の高い環境構築を実現するためにAnsibleを使用する。

## 2. 管理対象範囲

本リポジトリでは以下のレイヤーを管理対象とする。

- **OS設定**: 基本的なOSチューニング、セキュリティ設定（ファイアウォール、ユーザー管理等）。
- **Kubernetes基盤**: クラスターの構築・管理設定（Ingress Controllerのセットアップを含む）。
- **データベース設定**: MySQLデータベースの導入・設定（ホストOSに直接導入）。

※アプリケーションレイヤー（コンテナイメージ、マニフェスト等）は本プロジェクトの範囲外とし、各アプリケーションのリポジトリで管理する。

## 3. インフラ前提条件

- **プラットフォーム**: OCI (Oracle Cloud Infrastructure)
- **インフラ構成管理**: `infra-oci-terraform` プロジェクトによってプロビジョニングされたインスタンスを対象とする。
- **OS**: Ubuntu (LTS) を想定。

## 4. 技術要件

- **構成管理ツール**: Ansible
- **Webサーバ/リバースプロキシ**: Kubernetes Ingress (`ingress-nginx`)
- **コンテナオーケストレーション**: Kubernetes (k8s)
- **データベース**: MySQL (ホストOS直接インストール)
- **ソース管理**: GitHub

## 5. 機能要件

- **OS全般設定**:
  - システム共通設定（タイムゾーン、ロケール、ホスト名管理）。
  - パッケージマネージャーの最適化とパッケージの最新化。
  - ユーザー・グループ管理および権限設定（sudo等）。
  - セキュリティ基盤（SSHの堅牢化、ファイアウォール設定等）。
  - 共通で利用するシステムユーティリティの導入。
- **Kubernetes環境構築**:
  - シングルノード構成のk8sクラスター導入。
  - 必要なアドオン（CNI, Ingress Controller `ingress-nginx`等）のセットアップ。
  - Ingress Controller は外部から直接 HTTP/HTTPS (80/443) トラフィックを受信する構成。
- **データベース管理**:
  - MySQLサーバーの導入と初期設定（文字コード、接続制限等）。
  - スキーマ単位でのデータベース作成およびユーザー権限管理。
  - Kubernetes上のアプリケーションからの安全な接続許可設定。
- **ミドルウェア共通設定**: 各サービスで共通して利用するライブラリやランタイムの導入。

## 6. 運用・保守要件

- **再現性**: 全ての構成をAnsibleプレイブックとしてコード化し、環境の再構築を容易にする。
- **メンテナンス性**: 役割ごとにRoleを分割し、見通しの良いディレクトリ構成とする。
- **セキュリティ**:
  - SSH公開鍵認証によるアクセス。
  - 機密情報（パスワード、APIキー等）は Ansible Vault を使用して暗号化管理する。
