# 基本設計書 - infra-oci-ansible

## 1. システム構成

本システムは、OCI (Oracle Cloud Infrastructure) 上のインスタンスに対して、Ansible を用いて OS およびミドルウェアの構成管理を行う。

### 1.1. 管理構成

- **構成管理サーバー**: ローカル環境（または CI/CD 環境）
- **対象サーバー**: OCI 上の Compute Instance (Ubuntu 24.04 LTS)
- **接続経路**: OCI Bastion サービスを経由した SSH 接続

### 1.2. ソフトウェア構成

| カテゴリ | ソフトウェア/ツール | 用途 |
| :--- | :--- | :--- |
| 構成管理 | Ansible | OS・ミドルウェアの自動構築 |
| OS | Ubuntu 24.04 LTS | ベース OS |
| Web サーバ | Nginx | リバースプロキシ、SSL 終端 |
| 証明書管理 | Certbot (Let's Encrypt) | SSL 証明書の自動取得・更新 |
| コンテナ基盤 | Kubernetes (k8s) | アプリケーション実行基盤 |
| 秘匿情報管理 | Ansible Vault | パスワード、鍵情報の暗号化 |

## 2. ディレクトリ構成 (Ansible Role 構造)

Ansible のベストプラクティスに基づき、Role 単位で機能を分割する。

```text
/home/seiya/git/infra-oci-ansible/
├── group_vars/           # グループ別変数 (all.yml, k8s.yml 等)
├── host_vars/            # ホスト別変数
├── inventory/            # インベントリファイル (hosts.yml)
├── roles/                # 各役割の定義
│   ├── common/           # OS共通設定 (タイムゾーン, ロケール, パッケージ更新)
│   ├── security/         # セキュリティ設定 (SSH堅牢化, ファイアウォール)
│   ├── nginx/            # Nginx導入・設定
│   ├── certbot/          # SSL証明書取得
│   └── k8s_base/         # Kubernetes基盤 (Docker/containerd, kubeadm等)
├── playbooks/            # プレイブック実行ファイル
│   ├── site.yml          # 全体実行用
│   ├── setup_os.yml      # OS初期設定用
│   └── setup_k8s.yml     # k8s構築用
└── ansible.cfg           # Ansible設定ファイル
```

## 3. OS 設計

### 3.1. 基本設定

- **タイムゾーン**: `Asia/Tokyo`
- **ロケール**: `ja_JP.UTF-8`
- **パッケージ管理**: `apt` を使用し、最新状態を維持。

### 3.2. ユーザー・権限管理

- **管理ユーザー**: `infra-admin` (sudo 権限付与)
- **認証方式**: 公開鍵認証のみ許可 (パスワード認証禁止)

### 3.3. セキュリティ設定

- **SSH**:
  - Root ログイン禁止
  - デフォルトポートの変更（検討事項）
- **ファイアウォール (ufw/iptables)**:
  - 許可: 80(HTTP), 443(HTTPS), SSHポート
  - その他は原則拒否

## 4. ミドルウェア設計

### 4.1. Nginx

- **役割**: 複数のサービス（サブドメイン別等）へのルーティング。
- **設定管理**: `sites-available` / `sites-enabled` 形式でバーチャルホストを管理。
- **SSL**: Let's Encrypt を利用し、Ansible で自動更新タスクを登録。

### 4.2. Kubernetes 基盤

- **ランタイム**: `containerd`
- **ネットワーク (CNI)**: Flannel または Calico (要件に応じて選定)
- **Ingress**: Nginx Ingress Controller (ホストの Nginx とは別に、k8s 内で動作)

## 5. 秘匿情報の管理 (Ansible Vault)

以下の情報は `ansible-vault` を用いて暗号化し、リポジトリに含める。

- API キー、パスワード
- 証明書などの秘密鍵
- その他、機密性の高い変数

## 6. 運用設計

### 6.1. 実行フロー

1. `inventory/hosts.yml` に対象サーバーを定義。
2. `ansible-playbook` コマンドで対象の Role を適用。
3. OCI Bastion 経由で SSH 疎通ができることを前提とする。

### 6.2. メンテナンス

- 定期的な OS セキュリティパッチの適用（Ansible での一括実行）。
- Kubernetes のバージョンアップ手順の確立。
